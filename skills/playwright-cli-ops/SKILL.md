# Playwright-cli Ops

## 1. 适用场景

在以下场景触发：

1. 需要通过浏览器执行搜索。
2. 需要打开候选页面并提取正文。
3. 需要把页面内容导出为后续清洗输入。

不要在以下场景触发：

1. 本地知识已经足够，不需要联网。
2. 已经拿到页面正文，只需要做清洗或持久化。

## 2. 职责边界

本 skill 只负责“如何用 Playwright-cli 稳定地拿到网页内容”：

1. 优先使用官方 `playwright-cli` 命令。
2. 如果项目本地已安装 package，可使用 `npx --no-install playwright-cli`。
3. 优先做搜索，再做候选页筛选，再做页面打开与内容导出。
4. 记录标题、URL、站点、页面类型、命中原因。

本 skill 不负责：

1. 把网页内容清洗成最终知识文本。
2. LLM 分块。
3. Milvus 或本地文件持久化。

## 3. 使用规则

1. 始终优先调用官方 `playwright-cli` 命令。
2. 如果命令不存在，明确报错并提示安装 `npm install -g @playwright/cli@latest` 或在项目中本地安装。
3. 搜索时优先带上站点限定词、产品名、版本词和主题词。
4. 打开页面前先做候选页筛选，尽量排除导航页、搜索页、广告页、聚合页。
5. 抓取时保留标题、正文、代码块、表格、FAQ、步骤说明。
6. 失败时直接暴露命令失败，不要静默切换到别的抓取实现。

## 4. 输出要求

输出至少包含：

1. 本轮执行的 Playwright-cli 命令。
2. 候选页面列表。
3. 最终选中的页面。
4. 导出的原始页面内容或正文文本。
