# get-info-workflow

## 为什么需要这个 skill

qa-agent 在回答用户问题时，如果发现本地知识不足，需要从外部补库。但"补库"不是一步操作，而是一条多阶段流水线：前置检查 → 读取优先级 → 生成检索计划 → 搜索 → URL 分类 → 调度抓取 → 入库 → 更新优先级。如果让 qa-agent 自己编排这些步骤：

1. **qa-agent 会变得臃肿**：它既要回答问题，又要管搜索调度、基础设施检查、优先级更新，职责严重过载。
2. **编排逻辑无法复用**：其他场景（如用户主动要求"联网补充"）也需要同样的编排，如果写在 qa-agent 里就无法复用。

这个 skill 就是 qa-agent 和底层能力（web-research-ingest / content-cleaner / knowledge-persistence）之间的**编排层**，核心价值是：

- 前置健康检查与降级决策矩阵（Playwright/Milvus 不可用时如何降级）
- 检索计划生成（查询改写 + 站点优先级 + 时间窗口策略）
- 调度编排（搜索 → URL 分类 → 返回候选列表给 qa-agent）
