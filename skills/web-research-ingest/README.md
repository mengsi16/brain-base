# web-research-ingest

## 为什么需要这个 skill

搜索引擎的使用看似简单（输入关键词 → 拿结果），但在知识库补库场景下有大量策略性工作：

1. **时间窗口优先级**：用户问"最新资料"时，应该先搜近1个月的结果；问"历史版本"时，应该跳过近期直接搜对应时间段。盲目搜全量既浪费配额又淹没高质量结果。
2. **URL 分类**：搜索结果里混杂官方文档、社区博客、广告落地页，必须分类后才能决定抓取策略（official-doc 逐字保留 vs community 提炼知识点 vs discard 直接丢弃）。
3. **与抓取解耦**：搜索和抓取是两个独立操作，搜索只产出 URL 候选列表，抓取由 content-cleaner 并行完成。把两者混在一起会导致串行瓶颈。

这个 skill 就是"搜索引擎 → 候选 URL 列表"这条链路的**标准化封装**，核心价值是：

- Google 时间段高级搜索的分批策略（5 个时间窗口，近期优先，够用就停）
- URL 分类的两级逻辑（白名单快速路径 + LLM 综合判断）
- 只输出结构化候选列表，不触碰页面内容
