# content-cleaner-workflow

## 为什么需要这个 skill

外部网页抓取后拿到的原始 HTML 充满噪音（导航栏、广告、Cookie 提示等），直接入库会导致检索命中率极低。需要一个专门负责"单 URL → 清洗 → 落盘 → 入库"的标准化流水线，核心解决两个问题：

1. **清洗规则因来源而异**：官方文档必须逐字保留（禁止概括/缩写），社区内容需要提炼知识点（去营销、标注来源）。两种来源的清洗策略完全不同，不能混为一谈。
2. **完整性校验**：LLM 清洗时容易过度删减或凭空补写，必须有量化校验（长度比率 ≥50%、章节计数一致、无 LLM 补写痕迹）来兜底。

如果没有这个 skill，每个 agent 都要自己实现清洗+校验逻辑，规则会漂移且无法统一审计。
