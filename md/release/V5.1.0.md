# V5.1.0 Release Notes

## 总结

**从"多轮对话 Agent"升级为"自主规划、工具调度、意图循环的 Agentic RAG 系统"。**

核心变化：废弃 T46 三路分流（classify_plan → hop_planner → hop_executor），引入统一意图识别 Agent-Loop（intent_planner → intent_executor → intent_observer），LLM 自主调度 6 工具完成搜索/抓取/检索/PDF 解析。

---

## 核心变化（4 大新能力）

### 1. 统一意图识别 Agent-Loop（替代三路分流）

| | 旧版本（V5.0.0） | 新版本（V5.1.0） |
|---|---|---|
| 证据收集架构 | T46 三路分流：classify_plan → hop_planner → hop_executor，每跳独立决策 | T47 统一循环：intent_planner → intent_executor → intent_observer，observer 评估后决定继续或终止 |
| 终止条件 | hop_planner prompt 内嵌判断（不可靠） | **5 级代码层早退**：连错 ≥2 / 充分 / 超上限 / no_action / 继续 |
| 防死循环 | 依赖 LLM 自觉停止 | 代码层硬约束（iteration_count + max_iterations + consecutive_errors） |
| 状态管理 | 每跳独立 state，跨跳信息靠 prompt 拼接 | 统一 evidence_pool + visited_urls + last_intent_observation，observer 写完整结构供 planner 下跳读取 |
| 新增节点 | — | `extract_urls`（正则提取用户问题中的 URL）+ `url_pre_fetch`（浅抓 URL 内容作改写上下文）+ `merge_evidence`（evidence_pool → get_info_candidates 13 字段标准格式） |

### 2. TOOL_REGISTRY 工具补齐（4 → 6 工具）

| | 旧版本（V5.0.0） | 新版本（V5.1.0） |
|---|---|---|
| 工具数量 | 4 个：web_search / fetch_url / raw_text / local_search | **6 个**：新增 arxiv_pdf / github_raw |
| arxiv_pdf | 无；arxiv URL 只能走 fetch_url 拿 abs 页 HTML | fetch_binary 下载 PDF → SHA-256 去重 → MinerU 全文解析 → 返前 3000 字 markdown + raw_path 透传 |
| github_raw | raw_text 内部暗调 `_try_github`（host 自动路由，LLM 不可见） | **一等工具**：LLM 显式选择；抽 `_match_github_url` 共享 helper；比 fetch_url 快 5-10× |
| 工具选择 | LLM 只能从 4 个里选 | LLM 按 URL 特征自主判断：arxiv.org → arxiv_pdf，github.com/blob → github_raw，其余 → fetch_url |
| 串行守门 | 无；并行跑多个 MinerU 会 OOM | intent_executor 按 `parallel_ok=False` 分组 → 串行/并行双队列同时跑 |
| 网页抓取 | raw_text 内部 urllib 直抓（无反爬） | **全部统一走 Playwright**（web_fetcher 单例 + stealth + auto-scroll） |

### 3. 固化层全自动（无需手动反馈）

| | 旧版本（V5.0.0） | 新版本（V5.1.0） |
|---|---|---|
| 固化触发 | 依赖外部 Agent 发 `feedback confirmed` | `crystallize_answer` 节点根据 LLM 评分 **value_score ≥ 0.3 自动写入** |
| 外部依赖 | 需要 `feedback` / `resume` / `history` 三条命令配合 | **零外部依赖**；ask 一次调用完成全部闭环 |
| 用户体验 | 问答完还要记得发反馈，否则知识不积累 | 问答即积累，满意答案自动固化，下次秒返 |

### 4. brain-base-skill 外部调用手册重写

| | 旧版本（V5.0.0） | 新版本（V5.1.0） |
|---|---|---|
| 文档状态 | 描述过时的 claude-plugin 模式（`bin/brain-base-cli.py` + `claude -p --agent`） | 完全对齐 LangGraph CLI（`python -m brain_base.cli`） |
| 命令覆盖 | 11 个命令中 5 个不存在（exists/ingest-text/feedback/resume/history） | 9 个命令全部可执行 |
| 环境变量 | `BRAIN_BASE_CLAUDE_BIN`（不存在） | `BB_LLM_PROVIDER / BB_LLM_API_KEY / BB_DEEP_THINK_LLM` 等 |
| 架构描述 | claude plugin 三件套（qa-agent/get-info-agent/upload-agent） | LangGraph 8 子图 |
| 新增场景 | — | 场景 E：跨进程多轮对话（`ask --session`） |

## 架构对比

```
V5.0.0（T46 三路分流）:
  normalize → decompose → classify_plan → hop_planner → hop_executor
                              ↓                    ↓
                          direct_url           search_web
                              ↓                    ↓
                          fetch_url           fetch_extract
                              ↓                    ↓
                          merge → persist → PIPE2 → judge → answer

V5.1.0（T47 统一意图 Agent-Loop）:
  probe → crystallized_check ── hit ──→ answer → END
       → extract_urls → url_pre_fetch → normalize → decompose
       → intent_planner ⇄ intent_executor ⇄ intent_observer
              ↑                   ↓                   ↓
              └── 5 级早退 ← ────┘         evidence_pool
                                                ↓
       → merge_evidence → persist 流水 → PIPE2 → judge → answer
       → self_check → crystallize_answer（自动固化）→ END
```
